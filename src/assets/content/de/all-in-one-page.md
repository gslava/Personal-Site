## Inventory Management System

Ein System zur Lagerverwaltung, das Folgendes umfasste:

- Sammlung von Bestellungen aus mehr als 10 US-Marktplätzen (Amazon, Walmart, eBay, Sears, BestBuy und weitere) und Speicherung ihrer Informationen in einem einheitlichen Format
- Sammlung von Informationen über alle Sendungen aus sämtlichen Unternehmenslagern sowie aus Amazon-Fulfillment- und Walmart-Fulfillment-Lagern
- Analysen und Statistiken für jedes Produkt mit Parametern und Filtern, die vom Nutzer gesteuert werden
- Informationen über Produktbewertungen aus verschiedenen Marktplätzen
- Informationen über bevorstehende Produktlieferungen und die Lieferhistorie
- ein KI-Chat, in dem Nutzer Fragen zu den im System gespeicherten Informationen stellen konnten
- ein Dashboard mit Statistiken und einem Problembereich

### Technologien

- Java 17
- Spring Boot
- PostgreSQL
- ReactJS
- RabbitMQ

## Orders Processing Helper

Eine Anwendung, die eine nahezu vollständige Automatisierung des Workflows zur Bestellverarbeitung umfasste. Die Anwendung verfügte über einen Satz von Regeln, die Nutzer konfigurieren konnten, und auf Basis dieser Regeln wurden folgende Aktionen ausgeführt:

- Vorverarbeitung:
  - Ergänzung von Informationen, die für die Verarbeitung erforderlich waren, falls sie fehlten:
    - Gewicht
    - Abmessungen
    - Zollinformationen für internationale Bestellungen
    - Warnhinweise zum Inhalt (zum Beispiel, wenn das Produkt Lithiumbatterien enthielt)
    - Informationen für das Verpackungspersonal
    - weitere produktspezifische Informationen
- Bestellverarbeitung:
  - Auswahl der kosteneffizientesten Versandmethode
  - Erstellung von Versandlabel und Rechnung
  - Verarbeitung von PDF-Dateien für unterschiedliche Formate und Druckertypen
  - Übermittlung von Informationen an Lager

### Technologien

- Java 17
- Spring Boot
- PostgreSQL
- Vaadin
- RabbitMQ

## Amazon Ads App

Eine Anwendung, die Daten über Werbekampagnen und deren Performance über APIs sammelte und diese anhand vordefinierter Regeln konfigurierte. Die Anwendung verfügte außerdem über eine Benutzeroberfläche, in der KI mögliche Verbesserungen vorschlug und der Nutzer die Änderung freigeben, anpassen oder ablehnen konnte. Die Anwendung schlug Änderungen an Keywords, negativen Keywords und Cost-per-Click-Geboten sowohl auf Kampagnenebene als auch für jedes einzelne Keyword vor. Außerdem schlug sie die Erstellung von Kampagnen für Produkte vor, die noch keine Werbekampagnen hatten. Die Anwendung bezog Daten aus Helium10, der Amazon Ads API und aus Internetsuchen, die von einem KI-Agenten durchgeführt wurden.

### Technologien

- Java 17
- Spring Boot
- PostgreSQL
- Vaadin
- RabbitMQ

## Product Content Creation App

Dieses Projekt wurde zur Generierung marktplatzspezifischer Inhalte erstellt. Der Nutzer gab Produktcodes ein, für die Inhalte generiert werden sollten, und der Service erzeugte eine Produktbeschreibung mit allen für den jeweiligen Marktplatz erforderlichen Feldern gemäß dessen Regeln. Der Nutzer konnte die Datei herunterladen oder das Ergebnis bei Bedarf in der Benutzeroberfläche bearbeiten. Anschließend wurde das Produkt dem Marktplatz hinzugefügt.

## Customer Support Bot

Ein Kundensupport-Bot, der Nutzeranfragen automatisch im Rahmen der Unternehmensregeln und -richtlinien bearbeitete. Wenn eine Anfrage außerhalb der zulässigen Szenarien lag, eine Ausnahme darstellte oder eine zusätzliche Prüfung erforderte, leitete das System sie an den zuständigen Support-Mitarbeiter weiter. Die Lösung arbeitete auf Basis von Daten aus dem Inventory Management System und verarbeitete Anfragen aus mehreren Kanälen: Live-Chats, E-Mails und Marktplätze.

### Technologien

- Java 17
- Spring Boot
- PostgreSQL
