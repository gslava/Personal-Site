## Inventory Management System

Система для керування складськими запасами, яка включала:

- збір замовлень із 10+ маркетплейсів США (Amazon, Walmart, eBay, Sears, BestBuy та інших) і збереження їхньої інформації в уніфікованому форматі
- збір інформації про всі відправлення з усіх складів компанії, а також зі складів Amazon Fulfillment і Walmart Fulfillment
- аналітику та статистику по кожному товару з параметрами й фільтрами, якими керував користувач
- інформацію про відгуки на товари з різних маркетплейсів
- інформацію про майбутні поставки товарів та історію поставок
- AI-чат, у якому користувачі могли запитувати про інформацію, що зберігалася в системі
- dashboard зі статистикою та розділом проблем

## Orders Processing Helper

Застосунок, який включав майже повну автоматизацію workflow обробки замовлень. У застосунку був набір правил, які користувачі могли налаштовувати, і на основі цих правил виконувалися такі дії:

- Попередня обробка:
  - додавання інформації, необхідної для обробки, якщо її бракувало:
    - вага
    - розміри
    - митна інформація для міжнародних замовлень
    - попередження про вміст (наприклад, якщо товар містив літієві батареї)
    - інформація для пакувальників
    - інша специфічна для товару інформація
- Обробка замовлення:
  - вибір найбільш економічно вигідного способу доставки
  - генерація shipping label та invoice
  - обробка PDF-файлів для різних форматів і типів принтерів
  - надсилання інформації на склади

## Amazon Ads App

Застосунок, який збирав дані про рекламні кампанії та їхню ефективність через API і налаштовував їх відповідно до наперед визначених правил. Також застосунок мав UI для користувачів, де AI пропонував можливі покращення, а користувач міг підтвердити, змінити або відхилити запропоновану зміну. Застосунок пропонував зміни для keywords, negative keywords і cost-per-click ставок як на рівні кампанії, так і для кожного окремого keyword. Також він пропонував створення кампаній для товарів, які ще не мали рекламних кампаній. Застосунок брав дані з Helium10, Amazon Ads API та інтернет-пошуку, який виконував AI-агент.

## Product Content Creation App

Цей проєкт був створений для генерації контенту під конкретні маркетплейси. Користувач вводив коди товарів, для яких потрібно було згенерувати контент, а сервіс створював опис товару з усіма необхідними полями для конкретного маркетплейсу відповідно до його правил. ## Customer Support Bot

Бот служби підтримки, який автоматично обробляв запити користувачів у межах правил і політик компанії. Якщо запит виходив за межі дозволених сценаріїв, був винятком або потребував додаткової перевірки, система передавала його відповідному спеціалісту підтримки. Рішення працювало на основі даних з Inventory Management System і обробляло запити з кількох каналів: live chat, email та маркетплейсів.
