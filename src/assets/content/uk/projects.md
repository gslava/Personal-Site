# Inventory Management System
Система керування складськими запасами, яка включала в себе:
- збір замовлень із 10+ маркетплейсів США (Amazon, Walmart, eBay, Sears, BestBuy та інших) і збереження інформації про них в уніфікованому форматі;
- збір інформації про всі відправлення з усіх складів компанії, а також зі складів Amazon Fulfillment і Walmart Fulfillment;
- аналітику та статистику щодо кожного товару з параметрами й фільтрами, якими керував користувач;
- інформацію про відгуки на товари з різних маркетплейсів;
- інформацію про майбутні поставки товарів та історію поставок;
- AI-чат, у якому можна було запитати інформацію, що зберігалася в системі;
- dashboard зі статистикою та розділом із проблемами.

### Технології
- Java 17
- Spring Boot
- PostgreSQL
- ReactJS
- RabbitMQ

# Orders Processing Helper
Додаток, який включав у себе майже повну автоматизацію процесу обробки замовлень. Додаток мав набір правил, які користувачі могли налаштовувати, і відповідно до цих правил виконувалися такі дії:
- Pre-processing:
	- додавання інформації, необхідної для обробки, якщо вона була відсутня:
		- вага;
		- розміри;
		- інформація для митниці для international-замовлень;
		- попередження про вміст (наприклад, якщо товар містить lithium batteries);
		- інформація для пакувальників;
		- інша специфічна для товару інформація.
- Обробка замовлення:
	- вибір найбільш вигідного способу доставки;
	- створення shipping label і invoice;
	- обробка PDF-файлів під різні формати для різних принтерів;
	- відправка інформації на склади.

### Технології
- Java 17
- Spring Boot
- PostgreSQL
- Vaadin
- RabbitMQ

# Amazon Ads App
Програма, яка через API збирала дані про рекламні кампанії, їхню статистику та налаштовувала їх за заданими правилами. Програма також мала UI для користувача, де AI пропонував, що можна покращити, а користувач міг затвердити зміну, скоригувати її або відхилити. Програма пропонувала зміни щодо ключових слів, негативних ключових слів, ставки за клік як для кампанії, так і для кожного ключового слова окремо. Також вона пропонувала створення кампаній для товарів, для яких рекламні кампанії ще не були створені. Дані програма отримувала з Helium10, Amazon Ads API та пошуку в інтернеті за допомогою AI-агента.

### Технології
- Java 17
- Spring Boot
- PostgreSQL
- Vaadin
- RabbitMQ

# Product Content Creation App
Цей проєкт був створений для генерації контенту під конкретний маркетплейс. Користувач вводив коди товарів, для яких потрібно було згенерувати контент, і сервіс генерував опис товару із заповненими всіма полями під конкретний маркетплейс згідно з правилами цього маркетплейса. Користувач міг завантажити файл або відредагувати результат у UI, якщо це було потрібно. Після цього товар додавався на маркетплейс.

### Технології
- Java 17
- Spring Boot
- PostgreSQL
- ReactJS

# Customer Support Bot
Бот для підтримки клієнтів, який автоматично обробляв звернення користувачів у межах правил і політик компанії. Якщо запит виходив за межі дозволених сценаріїв, був винятковим або потребував додаткового розгляду, система передавала його відповідному спеціалісту служби підтримки. Рішення працювало на основі даних з Inventory Management System та обробляло звернення з різних каналів: live-чатів, електронної пошти та маркетплейсів.

### Технології
- Java 17
- Spring Boot
- PostgreSQL

# Advanced Web Scrapers
- збір даних про stock tickers та аналітики щодо них;
- автоматизація дій на сайтах, які не мали API:
	- додавання і редагування товарів;
	- завантаження звітів;
	- додавання інформації про відправлення;
	- внесення інформації для митного брокера;
	- відстеження новин щодо змін у законодавстві, які могли вплинути на бізнес;
	- створення модулів з API, щоб програма могла виконувати дії на сайті через Selenium для автоматизації роботи з маркетплейсами з обмеженим API;
	- captcha solver.

### Технології
- Java 17
- Spring Boot
- PostgreSQL
- Selenium / Selenide
